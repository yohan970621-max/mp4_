# v2.1 업데이트 적용 방법

이 업데이트는 기존 Render 데이터베이스, 계정, 제출 영상 링크를 유지합니다. 새 Blueprint나 데이터베이스를 만들지 마세요.

## 변경된 평가 방식

최종 점수는 아래 세 점수의 산술평균입니다.

- 평균 시청시간 점수: 33.3%
- 평균 시청지속비율 점수: 33.3%
- 클릭률 점수: 33.3%

각 분야의 원자료는 서로 단위가 다르므로, 해당 반에서 그 분야 1위 작품을 100점으로 환산합니다.

```text
분야 점수 = 작품의 분야 원자료 ÷ 분야 최고 원자료 × 100
최종 점수 = (시청시간 점수 + 지속비율 점수 + 클릭률 점수) ÷ 3
```

### 원자료 정의

```text
평균 시청시간
= 모든 반복 재생·재시청을 포함한 누적 시청시간
÷ 해당 작품을 클릭한 고유 평가자 수

평균 시청지속비율
= 평균 시청시간 ÷ 영상 길이 × 100

클릭률
= 작품을 한 번 이상 클릭한 고유 평가자 수
÷ 해당 작품의 전체 평가 가능 인원 × 100
```

되감아 같은 구간을 다시 본 시간과 영상을 다시 처음부터 본 시간도 누적됩니다. 따라서 평균 시청지속비율은 100%를 넘을 수 있습니다. 같은 평가자가 여러 번 클릭해도 클릭률에서는 한 명으로 계산합니다.

## GitHub에 업데이트 적용

1. `video-evaluation-web-v2.1.0-patch.zip`의 압축을 풉니다.
2. GitHub의 기존 `mp4_` 저장소 첫 화면으로 이동합니다.
3. `Add file` → `Upload files`를 누릅니다.
4. 압축을 푼 패치 폴더 **안의 파일과 폴더 전체**를 업로드 영역으로 끌어놓습니다.
5. 기존 파일과 이름이 같다는 안내가 나와도 그대로 진행합니다.
6. 아래의 `Commit changes`를 누릅니다.
7. Render에서 자동 배포가 시작될 때까지 기다립니다.

저장소 루트에 다음 파일이 보여야 합니다.

```text
server.js
render.yaml
package.json
migrations/002_cumulative_watch_and_composite_score.sql
lib/scoring.js
lib/stats.js
```

## Render 확인

기존 `video-evaluation-web` 서비스만 사용합니다.

1. Render에서 `video-evaluation-web`을 엽니다.
2. 새 커밋의 상태가 `Deploying`인지 확인합니다.
3. 자동 배포가 시작되지 않으면 `Manual Deploy` → `Deploy latest commit`을 누릅니다.
4. `Live`가 표시되면 사이트를 새로고침합니다.

서버 시작 시 데이터베이스 마이그레이션이 자동 실행됩니다. 기존 계정과 작품은 유지됩니다.

## 테스트 기록 삭제 권장

기존 테스트 시청기록은 최초 누적값으로 보존됩니다. 실제 평가 전에 관리자 화면에서 다음 작업을 하세요.

```text
관리자 → 시청기록 초기화 → RESET 입력 → 시청기록 전체 삭제
```

계정과 제출 작품은 삭제되지 않습니다.

## 업데이트 확인 테스트

1. 관리자 계정에서 평가를 진행 상태로 만듭니다.
2. 강사 계정으로 한 작품을 약 12초 봅니다.
3. 재생바를 뒤로 돌려 같은 구간을 다시 약 12초 봅니다.
4. 작품 목록 또는 관리자 상세화면에서 누적 시청시간이 약 24초로 증가하는지 확인합니다.
5. 영상을 다시 열어 추가 시청하면 이전 누적시간에 계속 더해지는지 확인합니다.

시청기록은 약 4초 간격으로 저장되므로 실제 재생시간과 몇 초 정도 차이가 날 수 있습니다.

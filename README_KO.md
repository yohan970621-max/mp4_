# YouTube 링크형 영상 시청시간 평가 시스템 v2.0

학생들이 각자 YouTube에 **일부공개**로 영상을 올리고 링크만 제출합니다. 학생과 강사는 각자 다른 PC·휴대전화에서 공개 웹주소에 접속해 자유롭게 작품을 선택하며, 사이트 내부 YouTube 플레이어에서 실제 재생된 고유 시청구간을 중앙 PostgreSQL 데이터베이스에 저장합니다.

## 기존 버전과 달라진 점

- 강사 PC에서 BAT 파일을 실행하지 않습니다.
- 모든 학생이 인터넷 웹주소 하나로 접속합니다.
- 영상 파일을 서버에 업로드하지 않습니다.
- 학생이 자신의 YouTube 일부공개 링크를 직접 제출합니다.
- 영상 저장·인코딩·스트리밍은 YouTube가 처리합니다.
- 사이트는 계정, 영상 ID, 시청구간, 순위 데이터만 저장합니다.

## 포함 기능

- 최초 접속 시 관리자·강사·학생 계정 설정
- 학생별 PIN 로그인
- 학생 본인의 YouTube 링크 제출·수정
- 제출 기간과 평가 기간 분리
- 사용자별 작품 순서 무작위 고정
- 제작자 본인 작품 자동 제외
- 사이트 내부 YouTube 플레이어 자유 시청
- 1배속 재생 고정
- 재생 중이며 현재 탭이 보이는 구간만 기록
- 일시정지·버퍼링·광고·건너뛴 구간 미집계
- 같은 구간 반복 재생 시 한 번만 집계
- 전체 평가자 평균 시청시간 또는 평균 시청률 순위
- 작품별 시청자 기록과 개인별 최장 시청 작품 확인
- CSV 내려받기
- 계정 수정·PIN 변경·활성화 관리
- 전체 시청기록 초기화

## 가장 빠른 사용 방법

`DEPLOY_RENDER_KO.md`를 따라 Render에 배포하세요. 이 프로젝트의 `render.yaml`이 웹서버와 PostgreSQL 데이터베이스를 함께 생성합니다.

배포가 끝나면 생성된 `https://...onrender.com` 주소에 접속합니다. 처음에는 `/setup` 화면이 열립니다.

1. 행사명 입력
2. 관리자 PIN 입력
3. 강사 PIN 입력
4. 학생 21명의 이름과 PIN 입력
5. 설정 완료
6. 학생들에게 웹주소와 개인 PIN 전달
7. 학생이 `내 작품`에서 YouTube 일부공개 링크 제출
8. 제출 완료 후 관리자가 `제출 마감` 및 `평가 진행`으로 변경
9. 학생과 강사가 자유롭게 영상 시청
10. 관리자 화면에서 순위 확인 및 CSV 저장

## 평가 계산

기본 순위는 다음 값입니다.

```text
전체 평가자 평균 시청시간
= 최소 유효 시청시간을 넘긴 모든 평가자의 고유 시청시간 합계
÷ 평가 가능한 전체 인원
```

- 제작자 본인은 분모와 기록에서 제외됩니다.
- 클릭하지 않은 평가는 0초입니다.
- 기본 최소 유효 시청시간은 10초입니다.
- 같은 1초 구간을 반복해서 봐도 한 번만 인정합니다.
- 영상 길이가 크게 다르면 관리자 설정에서 `전체 평가자 평균 시청률`로 변경할 수 있습니다.

## YouTube 업로드 조건

학생은 YouTube Studio에서 공개 상태를 **일부공개**로 설정해야 합니다.

- 비공개: 사이트에서 재생되지 않으므로 사용 불가
- 일부공개: 사용 가능
- 공개: 사용은 가능하지만 평가 전 외부에 노출될 수 있음

외부 웹사이트 삽입이 차단된 영상, 연령 제한 영상 또는 YouTube에서 재생을 제한한 영상은 사이트 안에서 재생되지 않을 수 있습니다. 학생은 제출 화면의 미리보기에서 반드시 재생 여부를 확인해야 합니다.

## 기록되는 데이터

- 이름과 역할
- PIN의 암호화 해시
- YouTube 영상 ID와 제목
- 영상별 클릭 세션
- 시청한 1초 구간
- 재생·정지·탐색·종료 등의 이벤트
- 비정상 접속 확인용으로 암호화한 IP 식별값

원본 영상 파일은 이 시스템에 저장하지 않습니다.

## 운영 한계

사이트는 영상이 실제로 재생 중이고 탭이 보이는지를 확인하지만, 시청자가 물리적으로 화면을 계속 바라보는지는 판별할 수 없습니다. 또한 개발자 도구를 적극적으로 조작하는 고의적인 공격을 완전히 막는 시험 감독 시스템은 아닙니다.

학생이 YouTube 앱이나 youtube.com에서 직접 본 시간은 기록되지 않습니다. 반드시 평가 사이트 안에 삽입된 플레이어로 봐야 합니다.

## 개발자용 로컬 실행

로컬 PostgreSQL이 준비된 경우:

```bash
npm ci
cp .env.example .env
npm run db:migrate
npm start
```

환경변수 `DATABASE_URL`과 `SESSION_SECRET`을 설정해야 합니다.
